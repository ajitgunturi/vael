**Step 1: Fix overflow in FinancialHealthSummaryCard at 400dp.**

The `_MetricBadge` widgets in the `Wrap` use `Row(mainAxisSize: MainAxisSize.min)` with fixed-width elements (3px bar + icon + text). At 400dp the badges overflow because the row doesn't wrap within the badge itself.

Fix approach -- make the card responsive to narrow widths:
- In `_HealthCard.build`, wrap the `Wrap` widget's children so text truncates if needed. Specifically:
  - In `_MetricBadge`, add `Flexible` wrapper around the `Text` widget so it can shrink with `overflow: TextOverflow.ellipsis`
  - Alternatively, reduce badge padding from `Spacing.sm` horizontal to `Spacing.xs` when width is constrained
- The simplest fix: In `_MetricBadge`, change the `Text` widget to be wrapped in a `Flexible` with `overflow: TextOverflow.ellipsis` on the Text:
  ```dart
  Flexible(
    child: Text(
      '$label: $value',
      overflow: TextOverflow.ellipsis,
      style: Theme.of(context).textTheme.labelSmall?.copyWith(fontWeight: FontWeight.w600),
    ),
  ),
  ```
- This ensures that at 400dp, if a badge text is too long it truncates rather than overflowing
