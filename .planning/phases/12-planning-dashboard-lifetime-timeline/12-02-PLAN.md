**1. Create `lib/features/planning/providers/timeline_provider.dart`:**

Define `TimelineNodeType` enum: `milestone`, `purchase`, `fiDate`

Define `TimelineNode` class:
- `type` (TimelineNodeType)
- `age` (int) — the age at which this event occurs
- `label` (String) — short label e.g. "1Cr", "Car", "FI"
- `statusColor` (TimelineNodeStatus enum: onTrack, atRisk, behind) — maps to green/amber/red
- `detailRoute` (String?) — route identifier for navigation (e.g. "milestone", "purchase", "fi")
- `referenceId` (String?) — milestone ID or goal ID for navigation

Create `timelineNodesProvider` — `FutureProvider.family<List<TimelineNode>, ({String userId, String familyId})>`:
- Watch milestoneListProvider: create nodes for each milestone with age, label = formatIndianNumber(targetAmountPaise ~/ 100) truncated to compact form (e.g., "1Cr", "50L"), status from MilestoneStatus mapping (onTrack->onTrack, ahead->onTrack, behind->behind)
- Watch purchase goals from GoalDao (goalCategory='purchaseGoal'): create nodes with age computed from targetDate - DOB, label = goal name truncated to 6 chars, status = onTrack (default, no status engine for purchases)
- Watch fiDefaultInputsProvider: if hasLifeProfile, create FI node with age = currentAge + yearsToFi (computed via FiCalculator.yearsToFi), label = "FI", status = onTrack
- Sort all nodes by age ascending
- Return empty list if no life profile

Create `currentAgeProvider` — `Provider.family<int?, ({String userId, String familyId})>` that returns current age from fiDefaultInputsProvider.currentAge if hasLifeProfile, else null.

**2. Create `lib/features/planning/widgets/timeline_painter.dart`:**

`TimelinePainter` — CustomPainter that renders:
- **Decade columns**: equal-width columns for each decade from age 20 to 80 (or dynamic range based on nodes). Vertical separator lines between decades. Decade labels at top ("20s", "30s", "40s", etc.)
- **Horizontal timeline axis**: solid line running through the middle of the canvas
- **Current age marker**: vertical dashed line at the current age position, with "Now" label and age number
- **Milestone/purchase nodes**: colored circles (radius 10) positioned by age on the timeline axis. Fill color from statusColor: green = `Color(0xFF4CAF50)`, amber = `Color(0xFFFFC107)`, red = `Color(0xFFF44336)`. Short label below each circle.
- **FI date marker**: diamond shape (rotated square) instead of circle, same color coding. "FI" label below.
- All text uses `TextPainter` with TextStyle from constructor params (theme-aware).
- Accept `List<TimelineNode>` and `int? currentAge` in constructor.
- Accept `onNodeTap` callback: `void Function(TimelineNode)?` — uses `hitTest` with Offset comparison to detect taps within 20px of a node center.

`TimelineHitTester` — companion `GestureDetector` wrapper that converts tap positions to node taps via the painter's stored node positions.

**1. Create `lib/features/planning/widgets/savings_waterfall_chart.dart`:**

`SavingsWaterfallChart` — `StatelessWidget` rendering a stacked horizontal bar:
- Constructor takes: `int incomePaise`, `int expensesPaise`, `List<AllocationAdvice> allocations`, `int unallocatedPaise`
- Renders a single horizontal bar segmented into:
  - Full bar width = income (100%)
  - Expenses segment (red — `ColorTokens.of(context).expense`)
  - Then for each allocation advice in order: segment colored by targetType:
    - "emergencyFund" → blue (`Color(0xFF2196F3)`)
    - "sinkingFund" → teal (`Color(0xFF009688)`)
    - "investmentGoal" → green (`ColorTokens.of(context).income`)
    - "opportunityFund" → purple (`Color(0xFF9C27B0)`)
  - Unallocated remainder (gray — `ColorTokens.of(context).onSurfaceVariant.withValues(alpha: 0.3)`)
- Each segment has a label below it showing the amount in compact format (e.g., "25K", "1.2L")
- Use `Row` of `Expanded(flex: segment_proportion)` containers with colored backgrounds and rounded ends
- Minimum flex of 1 for visible segments to prevent zero-width
- Height: 32px for the bar, labels below
- Title "Savings Waterfall" above the bar

**2. Create `lib/features/planning/widgets/cash_flow_mini_view.dart`:**

`CashFlowMiniView` — `StatelessWidget` that takes `List<DayProjection> projections` (next 7 days):
- Renders a compact vertical list of 7 rows
- Each row: date (day name, e.g., "Mon 25"), net flow for that day (sum of items: income positive, expense negative), running balance for first account
- Rows with alerts get a warning icon (`Icons.warning_amber`) and amber background tint
- Compact: each row is 36px height
